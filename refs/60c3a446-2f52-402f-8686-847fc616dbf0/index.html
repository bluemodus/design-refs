Proposal Design System

All components available for BlueModus proposal documents. To build a new proposal, start from proposal-template.html and reference this guide for component markup.

Reference
Print & PDF
The CSS includes a full @media print block that overrides all token values to light-mode equivalents. No separate print stylesheet is needed. The JS in each proposal's "Download PDF" button temporarily makes all pages visible, calls window.print(), then restores state.

Print notes:
— All CSS tokens automatically switch to light-mode values at print time.
— Each .page becomes display:block with a page break after it.
— Header, nav, and footer are hidden at print.
— Use .print-only for elements only visible in print (e.g., letterhead).
— Cards, callouts, and deliverable blocks are set to break-inside: avoid.

Page Shell
The nav sidebar is auto-generated from data-title and data-group attributes on each .page div. Groups become nav section labels; titles become nav links. Pagination state is managed by the JS block at the bottom of the template.

  <div class="eyebrow">Section Group</div>

  <p class="intro">Opening paragraph.</p>
